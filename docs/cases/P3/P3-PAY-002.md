## P3-PAY-002 — Pay multiple vendor invoices in one batch run

```yaml
id: P3-PAY-002
title: Run a payment batch covering multiple vendor invoices
goal: |
  Verify the application supports a batch / payment-run workflow:
  select multiple approved vendor invoices, choose a payment method,
  generate one set of payments (one check per vendor by default),
  and post them as a group.
roles:
  - Controller
flows:
  - vendor-to-asset
  - part-to-inventory
preconditions:
  - At least three approved vendor invoices exist across at least
    two vendors.
prerequisite_cases:
  - P3-AP-001
steps:
  - n: 1
    action: |
      Find the payment-run area. Filter approved invoices by due date
      (e.g., due within 7 days). Select all.
    expected: |
      A payment run is staged with one check / ACH per vendor (sum
      of selected invoices for that vendor).
  - n: 2
    action: |
      Confirm and post the run.
    expected: |
      Payments are issued. Each invoice is marked paid. GL impact:
      cash credited, AP debited, in one consolidated transaction set.
  - n: 3
    action: |
      Open the AP register. Verify each invoice shows paid and
      references the correct check / ACH ID.
    expected: |
      All invoices show paid. Cross-references resolve.
expected_overall: |
  Batch payment run pays multiple invoices cleanly.
pass_criteria: |
  All selected invoices marked paid AND one payment per vendor AND
  GL postings tie to the run total.
est_minutes: 10
negative_variants:
  - id: P3-PAY-002-N1
    title: Cannot include held invoices in the batch
    action: |
      Mark one approved invoice as "on hold," then run the batch
      filter.
    expected: |
      Held invoices are excluded from the staged run with a clear
      "X invoices on hold" indicator.
    pass_criteria: |
      Held invoices excluded.
  - id: P3-PAY-002-N2
    title: Cannot post run with insufficient cash account balance
    action: |
      Stage a payment run that exceeds the source bank account's
      balance.
    expected: |
      Posting is blocked or warns; overdraft requires explicit
      acknowledgement.
    pass_criteria: |
      Overdraft prevention is in place.
  - id: P3-PAY-002-N3
    title: Cannot duplicate a payment run already posted
    action: |
      After posting, attempt to re-post the same run.
    expected: |
      The action is unavailable; the posted run cannot be re-issued.
    pass_criteria: |
      Re-post refused.
```
